# travelmapping
Travel Destination Interest Mapping
Overview:
Travel Destination Interest Mapping is a simple yet effective travel recommendation system developed as part of a Discrete Structures course project. The system uses fundamental concepts from discrete mathematics—specifically sets and relations—to match tourists with destinations that align with their personal interests such as beaches, culture, food, and adventure.
Built with C++ and Object-Oriented Programming (OOP) principles, this project demonstrates how theoretical mathematical concepts can be applied to solve real-world problems in an elegant and understandable way.
Problem Statement
Tourists often struggle to find destinations that truly match their interests. Manual searching across multiple websites is time-consuming, and existing recommendation systems are either overly complex or lack personalization. There is a clear need for a simple, transparent, and educational model that can recommend travel destinations using mathematical logic.
This project addresses that gap by designing a user-friendly recommendation system based on discrete mathematical relations.
Objectives:
Design a travel recommendation system using discrete structures
Apply set theory and relations to a real-world problem
Implement the system using C++ and OOP principles
Recommend destinations based on user interests
Provide a simple, educational, and efficient solution
Methodology:
The system follows a clear, step-by-step process:
Define a set of destinations (e.g., Paris, Maldives, Tokyo)
Define a set of interests (e.g., beaches, culture, food, adventure)
Create a relation between destinations and interests
Build a Destination class in C++
Store destinations using vectors
Take interest input from the user
Compare user input with stored interests
Display matching destinations as output
System Flow:
User → Interest Input → Relation Matching → Recommendation Output
This clean architecture ensures fast, accurate, and easy-to-understand recommendations.
Results & Discussion:
The system successfully recommends relevant destinations when a valid interest is entered.
If an invalid or unmatched interest is entered, the system displays: “No destination found”.
The relation mapping between destinations and interests works correctly.
The project effectively demonstrates how discrete relations can be applied in a practical, real-world recommendation system.
Conclusion:
Discrete structures (sets & relations) were successfully applied to a real-world problem.
The C++ OOP implementation makes the system organized, reusable, and easy to extend.
The system provides accurate, fast, and user-friendly recommendations.
This project serves as an excellent educational example of turning theoretical concepts into practical software.
